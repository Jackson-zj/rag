# 历史会话功能概要设计

## Architecture

- Spring Boot 继续负责会话所有权校验和 PostgreSQL 持久化，新增本人会话标题更新接口。
- React 维护会话列表、当前会话、编辑状态和移动端展开状态，复用现有列表与消息 API。
- 新对话采用延迟创建：前端先进入空状态，首次提问时才调用创建接口。

## Data Flow

1. 登录后获取本人会话列表，并按本地保存的 Session ID 恢复选择，否则使用最新会话。
2. 切换会话时读取最近 10 轮消息，成功后原子更新当前会话和消息区域。
3. 首次提问时以问题摘要为标题创建会话，再发送 SSE 请求。
4. 重命名请求由后端验证所有权和标题规则，成功后更新前端列表。

## Decisions

- 不新增数据库表或迁移。
- 不增加前端测试框架；以 TypeScript 构建、Java 测试和浏览器端到端验证为主。
- 移动端使用历史按钮控制会话栏，桌面端固定显示。

## Risks

- 异步切换可能造成旧请求覆盖新状态，因此流式回答和历史加载期间禁用会话操作。
- 本地保存的会话可能已不存在，登录恢复必须回退到最新会话。
