# 历史会话功能需求

## Goal

为所有已登录用户提供仅限本人范围的历史会话列表、切换、新建和手工重命名功能，并在刷新或重新登录后恢复上次选择。

## Non-Goals

- 不提供会话删除。
- 不允许管理员查看或管理其他用户的会话。
- 不改变现有最近 10 轮消息加载规则。

## Functional Requirements

- 桌面端显示会话侧栏，移动端可展开或收起。
- 新对话在首次发送问题时才创建，标题取首问前 32 个字符。
- 会话可原地重命名，标题去除首尾空格且长度为 1 至 60 个字符。
- 流式回答期间禁止切换、新建和重命名。
- 使用按用户隔离的浏览器存储恢复上次选中的会话。

## Success Criteria

- 用户能切换并读取本人任一历史会话。
- 刷新后恢复上次会话，不同账号不串会话。
- 重命名持久化且不能越权。
- 桌面和移动端布局可用，现有聊天、SSE 和权限行为不回退。

## Assumptions

- 会话继续按创建时间倒序，不因继续聊天或重命名重新排序。
- 现有会话不迁移或删除；同名会话通过时间区分。
